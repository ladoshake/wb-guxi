# A股股息率排名（市值 > 1000 亿 · TTM / LFY 前 30）

筛选 A 股总市值超过 1000 亿元人民币的公司，按股息率分别取 **TTM（近 12 个月）** 与 **LFY（最近财年）** 排名的前 30 名，生成可交互的网页榜单。股息率根据历史分红明细**自行计算**，不依赖任何现成股息率字段。

## 功能特性

- **双榜单切换**：TTM 股息率排名 / LFY 股息率排名
- **LFY 表附加趋势列**：在「LFY 股息率」之后，额外列出**前年股息率**（LFY 财年 −1）与**大前年股息率**（LFY 财年 −2），口径与 LFY 一致，便于观察分红连续性
- **同财年多次分红累加**：年内若有年报 + 中期 + 特别分红，均按财年求和（如某家公司 FY2025 两次分红 5.6 + 4.4 = 10.0 元/10 股）
- **表头点击排序**、**隔行底色**、**悬停高亮**
- **数据刷新按钮**（需本地服务器，见下方「运行方式」）
- 纯前端渲染，数据为抓取日快照，可离线打开查看

## 数据口径

| 指标 | 计算方式 |
|---|---|
| **总市值筛选** | 腾讯 gtimg 实时行情「总市值」（含 H 股），阈值 > 1000 亿元 |
| **每股分红** | 巨潮历史分红明细的派息比例（元/10 股）÷ 10 = 元/股 |
| **TTM 股息率** | 近 12 个月内（除权日落入窗口）全部现金分红合计数 ÷ 当前总市值 × 100% |
| **LFY 股息率** | 最近一个完整财年（按分红年度取最大年）全部现金分红合计数 ÷ 当前总市值 × 100% |
| **前年 / 大前年股息率** | 口径同 LFY，分别对应 LFY 财年 −1、−2 的全部分红 |

> 运行日期为动态 `dt.date.today()`，因此 TTM 窗口与「最近财年」随运行日自动更新，每次刷新反映的是当日真实情况。

## 文件说明

| 文件 | 作用 |
|---|---|
| `fetch_data.py` | 抓取行情与分红、计算 TTM/LFY 股息率、生成 `data.json` |
| `build_html.py` | 读取 `data.json`，生成自包含网页 `index.html` |
| `serve.py` | 本地 HTTP 服务（`stdlib`，无需额外依赖），提供刷新接口 |
| `index.html` | 交付网页（数据内嵌，可直接双击打开） |
| `.gitignore` | 排除运行时缓存与本地目录 |

运行时生成的 `data.json`、`dividends_cache.json` 不纳入版本管理。

## 运行方式

环境要求：Python 3.10+，依赖 `akshare`、`pandas`、`requests`（建议使用虚拟环境）。

```bash
pip install akshare pandas requests

# 抓取数据 + 生成网页
python fetch_data.py
python build_html.py
```

### 仅查看榜单
直接双击打开 `index.html` 即可（使用内嵌的数据快照）。此时刷新按钮会提示需启动服务器。

### 实时刷新数据
启动本地服务器，再用浏览器访问 `http://localhost:8000`：

```bash
python serve.py
```

点击网页右上角「刷新数据」即可重跑抓取脚本并更新榜单。

| 接口 | 说明 |
|---|---|
| `GET /` | 打开 `index.html` |
| `GET /api/data` | 读取最新 `data.json` |
| `POST /api/refresh` | 重跑 `fetch_data.py` + `build_html.py` |

> 手动刷新复用已有分红缓存，仅实时重拉市值并重算（通常数十秒）；新披露的分红由每日定时任务（先清缓存全量重抓）入账。

## 数据来源

- 行情 / 总市值：腾讯 gtimg
- 股票列表、历史分红明细：akshare（巨潮）
